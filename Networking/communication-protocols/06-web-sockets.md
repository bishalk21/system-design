# Web Sockets

WebSockets is a communication protocol that provides full-duplex communication channels over a single TCP connection. It is designed to enable real-time, bidirectional communication between clients (such as web browsers) and servers, allowing for low-latency and efficient data exchange. WebSockets is commonly used in applications that require real-time updates, such as chat applications, online gaming, live streaming, and collaborative tools.
WebSockets operates on a different communication model compared to traditional HTTP. While HTTP is a request-response protocol where the client initiates a request and the server responds, WebSockets allows for continuous, bidirectional communication between the client and server without the need for repeated requests. This makes it ideal for scenarios where real-time updates are required, as it eliminates the overhead of establishing new connections and sending multiple HTTP requests.
The WebSocket protocol starts with an HTTP handshake, where the client sends an HTTP request to the server to upgrade the connection to a WebSocket. If the server supports WebSockets, it responds with an HTTP 101 Switching Protocols status code, indicating that the connection has been upgraded. Once the handshake is complete, the client and server can exchange messages in real-time over the established WebSocket connection.
WebSockets uses a simple message-based communication model, where messages can be sent in either direction at any time. Messages can be in text or binary format, and the protocol provides mechanisms for framing messages, handling fragmentation, and managing connection state. WebSockets also includes support for ping/pong messages to keep the connection alive and detect if the other party is still responsive.
Overall, WebSockets is a powerful protocol for enabling real-time communication in web applications, providing low latency and efficient data exchange between clients and servers. It is widely supported by modern browsers and web servers, making it a popular choice for applications that require real-time updates and interactive features. However, it is important to note that WebSockets may not be suitable for all types of applications, especially those that do not require real-time communication or can tolerate some level of latency, such as traditional web browsing or static content delivery. In such cases, alternative protocols like HTTP may be more appropriate.

- How WebSockets works:
  - The client initiates a WebSocket connection by sending an HTTP request to the server with an "Upgrade" header, indicating that it wants to upgrade the connection to a WebSocket.
  - If the server supports WebSockets, it responds with an HTTP 101 Switching Protocols status code, indicating that the connection has been upgraded to a WebSocket.
  - Once the handshake is complete, the client and server can exchange messages in real-time over the established WebSocket connection. Messages can be sent in either direction at any time, and can be in text or binary format.
  - The protocol provides mechanisms for framing messages, handling fragmentation, and managing connection state. It also includes support for ping/pong messages to keep the connection alive and detect if the other party is still responsive.
- Use cases: Chat applications, online gaming, live streaming, collaborative tools, real-time dashboards, and any applications that require real-time updates and interactive features. WebSockets is especially beneficial in scenarios where low latency and efficient data exchange are critical for a seamless user experience.
